# T.PDT.since()メソッドの返り値



## メタ情報

- **公開日**: 2026-06-10
- **ソース**: [Hail2u](https://hail2u.net/blog/temporal-plaindatetime-since-return-value)




## 本文

<p>ある日付が今より何年前かを知るため、TemporalオブジェクトのPlainDateTimeオブジェクトで<code>since()</code>メソッドを使い、以下のように書いて失敗する。2つも間違えているようだ。あまり関係ないがPが大文字だったり小文字だったりして間違える。</p>

<pre><code class="language-javascript">const then = Temporal.PlainDateTime.from(&quot;2020-01-01&quot;);
const now = Temporal.Now.plainDateTimeISO();

console.log(then.since(now).years); // 0</code></pre>

<p>6年半くらい前なので<code>-6.4</code>になると思い込んでいたが、Durationオブジェクトはそういうものではないらしい。更に<code>since()</code>メソッドで返ってくるDurationオブジェクトは<a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Temporal/PlainDateTime/since#options">既定で最大単位が日</a>になっており、年や月は持っておらず、<code>-6</code>にもならない。365日以上前かどうかと調べても良いが、せっかく減らせた魔法の数字を使わなくてはならない。</p>

<pre><code class="language-javascript">const then = Temporal.PlainDateTime.from(&quot;2020-01-01&quot;);
const now = Temporal.Now.plainDateTimeISO();

console.log(then.since(now, {
  smallestUnit: &quot;years&quot;,
}.years); // -6</code></pre>

<p>第二引数で最小単位を年にすると簡単そうだ。これで<code>years</code>プロパティーが負の整数ならば1年以上前になる(364日前では0になるし、未来の日付なら正の整数になる)。<code>since()</code>メソッドと<code>until()</code>メソッドがあやふやなので気を付けたい。</p>

<pre><code class="language-javascript">const then = Temporal.PlainDateTime.from(&quot;2020-01-01&quot;);
const oneYearAgo = Temporal.Now.plainDateTimeISO().subtract({
  years: 1
});

console.log(then.since(oneYearAgo.sign); // -1</code></pre>

<p><code>subtract()</code>メソッドを使って1年前の日付を作り、<code>sign</code>プロパティーを見るのも悪くなさそうだ。こちらの方が融通が利くかもしれない。<code>compare()</code>メソッドでも良いが、直感的でないコードになり、かつ長くなる。</p>
